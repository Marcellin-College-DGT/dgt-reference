tkinter widget reference

A guide to the building blocks of a Python GUI — what each widget is for, how to use it, and how it earns NCEA credit.

This guide is a reference you can return to while you build. Every widget gets a page with the same layout, so once you learn the shape you can scan any page fast. Every code example is a complete program — copy it, run it, then change it. The demos are where the real learning is.

Where the levels sit. Everything in the core pages is reachable at Level 2 (AS91896). Two of the named advanced techniques there are responding to events generated by a GUI and using a non-core library — and a tkinter program gives you both at once. Pages with an interactive widget also carry a Level 3 lift (AS91906): the same idea moved into a class you define, with file persistence and validation. The Year 13 widgets go further, and the capstone shows several complex techniques working together.

Classic tk vs themed ttk. tkinter has two families of widget. The classic ones (tk.Button, tk.Label…) are simple and take styling options like bg and fg directly. The themed ones (ttk.Button, from from tkinter import ttk) look more like native desktop controls, but are styled differently — they do not accept bg/fg; you change their look through a ttk.Style object instead.

This guide uses classic tk widgets by default because they are simpler to learn, and reaches for ttk where there is no classic version (Combobox, Treeview, Notebook). Appearance is not assessed in NCEA, so either family is fine — but where a widget exists in both, its page shows you the comparison. Note that some widgets are tk-only (Text, Listbox, Canvas, Menu) and a few are ttk-only.

Foundationsthe machinery every widget assumes

The window + mainloopFoundations

Every program has one main window, and one line that keeps it running.

What it is

The main window is created with tk.Tk() and is named root by convention. The last line of the program is always root.mainloop(), which hands control to tkinter and keeps the window open and responding to the user.

Why it matters

Nothing appears until there is a window, and nothing reacts until mainloop() runs. Every widget is created "inside" the window (or inside a Frame that is inside the window).

Minimal example

import tkinter as tk

root = tk.Tk()
root.title("My program")
root.geometry("320x200")   # width x height in pixels
root.mainloop()            # always the last line

Key points

  • tk.Tk() — call it once, for the one main window, named root.
  • .title(...) and .geometry("WxH") set the title bar and size.
  • mainloop() goes last and runs until the window is closed.

Common gotcha

Forget mainloop() and the program runs and exits instantly with no window. For a second window use Toplevel, not another tk.Tk().

pack & gridFoundations

Layout managers decide where each widget sits.

What it is

A widget is invisible until you "manage" its position. pack stacks widgets (top, bottom, left, or right); grid places them in rows and columns, which suits forms.

Why it matters

Choosing the right manager makes a layout tidy and predictable. Most programs use grid for forms and pack for simple stacks.

Minimal example

import tkinter as tk

root = tk.Tk()
tk.Label(root, text="Name:").grid(row=0, column=0)
tk.Entry(root).grid(row=0, column=1)
tk.Button(root, text="OK").grid(row=1, column=0, columnspan=2)
root.mainloop()

Key points

  • pack(side=, fill=, expand=, padx=, pady=)
  • grid(row=, column=, columnspan=, sticky=, padx=, pady=)
  • sticky="ew" stretches a widget across its cell.

Common gotcha

Do not mix pack and grid in the same container — the program will hang. Use one manager per container. (You can pack a Frame, then grid inside it — see the Frame page.)

StringVar / IntVarFoundations

Special variables the GUI watches and stays in sync with.

What it is

Control variables that tkinter keeps linked to a widget. Change the variable and the widget redraws; the user edits the widget and the variable updates.

Why it matters

They save you reaching into widgets directly, and they let two widgets share one value — a Label can mirror an Entry just by sharing its variable.

Minimal example

import tkinter as tk

root = tk.Tk()
name = tk.StringVar()
tk.Entry(root, textvariable=name).pack()
tk.Label(root, textvariable=name).pack()   # mirrors the Entry live
root.mainloop()

Key points

  • tk.StringVar(), tk.IntVar(value=0), tk.BooleanVar().
  • var.get() reads, var.set(...) writes.
  • Attach with textvariable= (Entry/Label) or variable= (Checkbutton/Radiobutton).

Common gotcha

StringVar.get() returns a string even for numbers; IntVar.get() returns an int. Choose the variable type that matches the data.

Event bindingFoundations

Making the program react to the user.

What it is

Two ways to run your code in response to the user. Buttons use command=; other widgets, keys, and the mouse use .bind("<event>", handler).

Why it matters

This is the heart of the Level 2 standard — responding to events generated by a GUI. Without it a window just sits there.

Minimal example

import tkinter as tk

root = tk.Tk()
entry = tk.Entry(root)
entry.pack()

def on_enter(event):
    print("You pressed Enter:", entry.get())

entry.bind("<Return>", on_enter)
tk.Button(root, text="Go", command=lambda: print("clicked")).pack()
root.mainloop()

Key points

  • command=my_func — pass the function (no brackets).
  • widget.bind("<Return>", handler) — handler receives an event.
  • Virtual events like <<ListboxSelect>> fire on selection changes.

Common gotcha

command=my_func() with brackets calls the function immediately instead of on click. And bind handlers take an event argument while command callbacks do not — a frequent source of "takes 0 arguments" errors.

Core widgetsYear 12 baseline · Level 2

LabelL2

Shows a piece of text (or an image) to the user.

What it is

A display area for a short piece of text or an image. A Label shows information to the user — it does not collect anything from them.

When to reach for it

A title, a prompt beside an Entry, a status line, or a result after a calculation. Not for input — for typing use Entry or Text; for a clickable action use Button.

Minimal example

import tkinter as tk

root = tk.Tk()
tk.Label(root, text="Kia ora!").pack(padx=20, pady=20)
root.mainloop()

tk vs ttk

Both families have a Label. Classic tk.Label takes colours directly; ttk.Label looks more native but is coloured through a ttk.Style.

Classic tk

tk.Label(root, text="Kia ora", fg="white", bg="#1D9E75")

Themed ttk

from tkinter import ttk
ttk.Label(root, text="Kia ora")   # no fg/bg here; colour via ttk.Style

Key options

  • text — the words to show.
  • textvariable — link a StringVar so it updates itself.
  • font, fg, bg (classic tk).
  • wraplength — pixel width at which long text wraps.

Events & value

A Label is output-only — it does not respond to events. Change what it shows with label.config(text="...") or by linking a StringVar and calling var.set(...).

Common gotcha

To update a Label live you must use .config() or a StringVar — reassigning the ordinary Python variable does nothing on screen. And it will not wrap long text unless you set wraplength.

ButtonL2Level 3 lift

Runs a function when clicked.

What it is

A clickable button that calls a function you write.

When to reach for it

Any action — submit, add, calculate, open a window. Not for choosing options; use Radiobutton or Checkbutton for that.

Minimal example

import tkinter as tk

root = tk.Tk()
status = tk.Label(root, text="Not clicked yet")
status.pack(padx=20, pady=10)

def greet():
    status.config(text="Hello!")

tk.Button(root, text="Greet", command=greet).pack(pady=10)
root.mainloop()

tk vs ttk

Both families have a Button. ttk.Button looks more native; its colour is set through a ttk.Style, not bg.

Classic tk

tk.Button(root, text="Save", bg="#1D9E75", command=greet)

Themed ttk

from tkinter import ttk
ttk.Button(root, text="Save", command=greet)   # no bg=; style via ttk.Style

Key options

  • text — the label on the button.
  • command — the function to call (no brackets).
  • state — "normal" or "disabled".

Events & value

Use command= for clicks. To pass an argument, use a lambda: command=lambda: choose("red"). A button has no value to read.

Common gotcha

Write command=greet, not command=greet(). In a loop, capture the loop value with a default argument: lambda c=colour: choose(c), or every button sends the last value.

Practice task

Level 3 lift

In a class, each button's command is a method — command=self.save, command=self.add. Organising handlers as methods of a class you defined is the structured, well-organised design Level 3 rewards.

EntryL2Level 3 lift

A single-line box the user types into.

What it is

A single-line text box the user types into. It is the most common way a tkinter program takes input from the user.

When to reach for it

One line of text or a number — a name, a search term, a score. For several lines use Text; for fixed options use Combobox or Radiobutton.

Minimal example

import tkinter as tk

root = tk.Tk()
name_var = tk.StringVar()
entry = tk.Entry(root, textvariable=name_var)
entry.pack(padx=20, pady=10)

def show():
    print("You typed:", name_var.get())

tk.Button(root, text="Show", command=show).pack(pady=10)
root.mainloop()

tk vs ttk

Both families have an Entry and they are used almost identically; ttk.Entry just looks more native.

Classic tk

tk.Entry(root, textvariable=name_var)

Themed ttk

from tkinter import ttk
ttk.Entry(root, textvariable=name_var)

Key options

  • textvariable — a StringVar to read or set the box.
  • width — width in characters.
  • show="*" — hide text, for passwords.
  • state="disabled" — read-only.

Events & value

Read it with name_var.get(), usually inside a button's command. Respond live by binding a key: entry.bind("<Return>", handler).

Common gotcha

.get() always returns a string, even if the user typed 42. Do maths on it without int() or float() and you get bugs or a crash — which is why validation matters.

Practice task

One Entry and one Button: clicking shows the typed text in a Label. Then extend it so an empty box shows a messagebox warning instead.

Level 3 lift

Move it into a class and add validation and persistence. This whole program runs as-is:

import tkinter as tk
from tkinter import messagebox

class NameApp:
    def __init__(self, root):
        self.var = tk.StringVar()
        tk.Entry(root, textvariable=self.var).pack(padx=20, pady=10)
        tk.Button(root, text="Save", command=self.save).pack(pady=10)

    def save(self):
        name = self.var.get().strip()
        if not name:                                       # validation
            messagebox.showwarning("Oops", "Please enter a name.")
            return
        with open("names.txt", "a", encoding="utf-8") as f:  # persistence (utf-8 for macrons)
            f.write(name + "\n")

root = tk.Tk()
NameApp(root)
root.mainloop()

That program shows a GUI, event handling, a class you defined, validation, and writing to persistent storage — four named complex techniques together.

TextL2Level 3 lift

A multi-line text box.

What it is

A box that holds many lines of text, unlike Entry which holds one.

When to reach for it

Notes, descriptions, messages — anything multi-line. For a single line use Entry.

Minimal example

import tkinter as tk

root = tk.Tk()
textbox = tk.Text(root, height=8, width=40, wrap="word")
textbox.pack(padx=10, pady=10)
textbox.insert("1.0", "Start here.")   # "1.0" = line 1, column 0

def show():
    print(textbox.get("1.0", tk.END))

tk.Button(root, text="Print", command=show).pack(pady=6)
root.mainloop()

tk vs ttk

Text is tk-only — there is no ttk.Text. Use tk.Text.

Key options

  • height, width, wrap="word".
  • insert(index, text), get(start, end), delete(start, end).
  • Indexes are strings like "1.0" and the constant tk.END.

Events & value

Read everything with textbox.get("1.0", tk.END). Bind <KeyRelease> to react as the user types — for a live word count, say.

Common gotcha

Indexes are strings ("1.0" means line 1, column 0), not numbers. And get("1.0", tk.END) includes a trailing newline — strip it if it matters.

Practice task

FrameL2Level 3 lift

An invisible container that groups widgets.

What it is

A container with no appearance of its own that groups other widgets so you can lay a window out in tidy sections.

When to reach for it

Organising a window into regions (a toolbar, a sidebar, a form), building a reusable component, or combining pack and grid in one window — each in its own Frame.

Minimal example

import tkinter as tk

root = tk.Tk()
left = tk.Frame(root)
left.pack(side="left", padx=10, pady=10)
tk.Button(left, text="A").pack()
tk.Button(left, text="B").pack()
root.mainloop()

tk vs ttk

Both families have a Frame. tk.Frame takes a border directly; ttk.Frame uses padding and is styled through ttk.Style.

Classic tk

tk.Frame(root, bd=1, relief="solid")

Themed ttk

from tkinter import ttk
ttk.Frame(root, padding=8)   # no bd/relief; border via ttk.Style

Key options

  • bd and relief="solid" give a visible border (classic tk).
  • padx, pady for internal spacing.
  • Use pack or grid inside it.

Events & value

None — a Frame is structural. Its job is layout, not interaction.

Common gotcha

Never mix pack and grid in the same container — tkinter will hang with no error. But each Frame is its own container, so you give each one the manager that fits: pack a toolbar Frame across the top, grid a form Frame's fields below. That is not a workaround — it is how the two managers are meant to be combined.

import tkinter as tk

root = tk.Tk()
toolbar = tk.Frame(root)
toolbar.pack(side="top", fill="x")        # toolbar: linear -> pack
tk.Button(toolbar, text="New").pack(side="left")

form = tk.Frame(root)
form.pack(side="top")
tk.Label(form, text="Name:").grid(row=0, column=0)   # form: rows/cols -> grid
tk.Entry(form).grid(row=0, column=1)
root.mainloop()

Subclass Frame to make a reusable component — a "labelled entry" you build once and drop in anywhere. Defining your own class like this is a named complex technique. This whole program runs as-is:

import tkinter as tk

class LabelledEntry(tk.Frame):
    def __init__(self, parent, caption):
        super().__init__(parent)
        tk.Label(self, text=caption).pack(side="left")
        self.var = tk.StringVar()
        tk.Entry(self, textvariable=self.var).pack(side="left")

root = tk.Tk()
LabelledEntry(root, "First:").pack(pady=4)
LabelledEntry(root, "Last:").pack(pady=4)
root.mainloop()
CheckbuttonL2Level 3 lift

An on/off (ticked/unticked) option.

What it is

A box the user ticks or unticks, backed by a variable you read to find its state.

When to reach for it

Independent yes/no settings where several can be on at once. For one-of-many, use Radiobutton.

Minimal example

import tkinter as tk

root = tk.Tk()
cheese = tk.BooleanVar()
tk.Checkbutton(root, text="Cheese", variable=cheese).pack(padx=20, pady=10)

def show():
    print("Cheese:", cheese.get())

tk.Button(root, text="Check", command=show).pack(pady=6)
root.mainloop()

tk vs ttk

Both families have a Checkbutton with the same variable model; ttk.Checkbutton looks more native.

Classic tk

tk.Checkbutton(root, text="Cheese", variable=cheese)

Themed ttk

from tkinter import ttk
ttk.Checkbutton(root, text="Cheese", variable=cheese)

Key options

  • variable — a BooleanVar (or IntVar).
  • text — the option label.
  • onvalue / offvalue to customise stored values.

Events & value

Read each with var.get() (True/False), or pass command= to react the moment it is toggled.

Common gotcha

Each Checkbutton needs its own variable. Share one variable across several and they all move together.

RadiobuttonL2Level 3 lift

Pick exactly one option from a group.

What it is

A set of mutually exclusive options. The trick is that every button in the group shares the same variable.

When to reach for it

A one-of-many choice with a few options. With many options a Combobox is tidier.

Minimal example

import tkinter as tk

root = tk.Tk()
size = tk.StringVar(value="Medium")
for label in ("Small", "Medium", "Large"):
    tk.Radiobutton(root, text=label, variable=size, value=label).pack(anchor="w")

def show():
    print("Picked:", size.get())

tk.Button(root, text="Confirm", command=show).pack(pady=6)
root.mainloop()

tk vs ttk

Both families have a Radiobutton with the same shared-variable model; ttk.Radiobutton looks more native.

Classic tk

tk.Radiobutton(root, text="Small", variable=size, value="Small")

Themed ttk

from tkinter import ttk
ttk.Radiobutton(root, text="Small", variable=size, value="Small")

Key options

  • variable — the shared variable for the group.
  • value — the value this button sets (unique per button).

Events & value

Read the shared variable with size.get(). React on change with command= or var.trace_add("write", ...).

Common gotcha

Every button in the group must share one variable, each with a distinct value — that is exactly what makes them exclusive. Separate variables = independent buttons.

ttk.ComboboxL2Level 3 lift

A dropdown to pick one value.

What it is

A dropdown menu (from ttk) the user picks one value from — the tidy alternative to a long row of Radiobuttons.

When to reach for it

Choosing one from a longer list. If the user should type freely, use Entry.

Minimal example

import tkinter as tk
from tkinter import ttk

root = tk.Tk()
choice = tk.StringVar()
combo = ttk.Combobox(root, textvariable=choice,
                     values=["DGT", "Hospitality", "MTI"], state="readonly")
combo.pack(padx=20, pady=10)
combo.current(0)

def show():
    print("Chose:", choice.get())

tk.Button(root, text="Show", command=show).pack(pady=6)
root.mainloop()

tk vs ttk

Combobox is ttk-only — there is no classic tk.Combobox. Always import it with from tkinter import ttk.

Key options

  • values — the list of choices.
  • textvariable — read the selection.
  • state="readonly" — pick only, no typing.
  • current(i) — preselect by index.

Events & value

Read with choice.get(). React when the user picks with combo.bind("<<ComboboxSelected>>", handler).

Common gotcha

It lives in ttk, not base tkinter. Without state="readonly" the user can type anything into it.

Level 3 lift

Populate values from a data source — a list read from a file or built from records — rather than hard-coding them.

ListboxL2Level 3 lift

A scrollable list of items the user can select from.

What it is

A box showing a scrollable list the user can select from, backed by a collection you add to and remove from as the program runs.

When to reach for it

Choosing from a list that changes — files, names, tasks. A small fixed set is cleaner as Radiobutton or Combobox; free text is an Entry.

Minimal example

import tkinter as tk

root = tk.Tk()
listbox = tk.Listbox(root)
for item in ("Apples", "Kumara", "Bread"):
    listbox.insert(tk.END, item)
listbox.pack(padx=20, pady=10)
root.mainloop()

tk vs ttk

Listbox is tk-only — there is no ttk.Listbox. For a multi-column table instead, see ttk.Treeview.

Key options

  • selectmode — "single", "extended", or "multiple".
  • height — rows shown.
  • insert(tk.END, item) / delete(index).
  • get(0, tk.END) — read the whole list.

Events & value

Find the selection with curselection() (returns indices), then listbox.get(index). React with <<ListboxSelect>>. Adding and removing items is "modifying data in a collection" — itself an advanced technique.

Common gotcha

curselection() returns a tuple of indices that can be empty. Index into it without checking and the program crashes. A Listbox stores display strings, not your objects — keep a parallel list if items carry more data.

Practice task

Level 3 lift

Move the list into a class that loads items from a file at startup and saves on every change. This whole program runs as-is:

import tkinter as tk

class TodoApp:
    FILE = "todo.txt"

    def __init__(self, root):
        self.listbox = tk.Listbox(root)
        self.listbox.pack(padx=20, pady=10)
        tk.Button(root, text="Add 'Task'",
                  command=self.add).pack(pady=4)
        self.load()                          # fill from the file at startup

    def add(self):
        self.listbox.insert(tk.END, "Task")
        self.save()

    def load(self):
        try:
            with open(self.FILE, encoding="utf-8") as f:
                for line in f:
                    self.listbox.insert(tk.END, line.strip())
        except FileNotFoundError:
            pass                             # first run: no file yet

    def save(self):
        with open(self.FILE, "w", encoding="utf-8") as f:
            for item in self.listbox.get(0, tk.END):
                f.write(item + "\n")

root = tk.Tk()
TodoApp(root)
root.mainloop()

A GUI, a class you defined, persistent storage, and collection handling — several complex techniques in one program.

Scale / SpinboxL2Level 3 lift

Enter a number within a range.

What it is

Two ways to enter a bounded number: a Scale is a slider; a Spinbox is a box with up/down arrows.

When to reach for it

Bounded numeric input — volume, quantity, difficulty. For free numbers use an Entry with validation.

Minimal example

import tkinter as tk

root = tk.Tk()
tk.Scale(root, from_=0, to=100, orient="horizontal").pack(padx=20, pady=10)

qty = tk.IntVar(value=1)
tk.Spinbox(root, from_=1, to=10, textvariable=qty).pack(pady=10)
root.mainloop()

tk vs ttk

Both families have these. The notable difference: ttk.Scale has no built-in number display, so you show the value yourself with a Label.

Classic tk

tk.Scale(root, from_=0, to=100, orient="horizontal")   # shows its value

Themed ttk

from tkinter import ttk
ttk.Scale(root, from_=0, to=100, orient="horizontal")  # no number shown

Key options

  • from_, to — the range.
  • orient — "horizontal" / "vertical" (Scale).
  • command — Scale callback (receives the value).
  • textvariable — an IntVar for the Spinbox.

Events & value

The Scale command callback receives the value as a string. The Spinbox with an IntVar gives you an int from get().

Common gotcha

The Scale callback value is a string like "37.0" — convert it with int(float(value)) before doing maths.

Level 3 lift

Recompute and update the interface live as the value changes (e.g. a price total), clamping and validating the input so the program stays robust.

messageboxL2

Pop-up info, warnings, and yes/no questions.

What it is

Ready-made dialogs from tkinter.messagebox for telling the user something or asking a simple question.

When to reach for it

Reporting a validation error, confirming a destructive action, or showing a result.

Minimal example

import tkinter as tk
from tkinter import messagebox

root = tk.Tk()

def ask():
    if messagebox.askyesno("Delete", "Are you sure?"):
        messagebox.showinfo("Done", "Deleted!")
    else:
        messagebox.showwarning("Cancelled", "Nothing happened.")

tk.Button(root, text="Delete", command=ask).pack(padx=20, pady=20)
root.mainloop()

tk vs ttk

messagebox is a dialog module, not a themed widget — there is no ttk version. It already uses native-looking dialogs.

Key options

  • showinfo, showwarning, showerror.
  • askyesno / askokcancel — return True/False.

Events & value

The ask* functions return a value (True/False) you branch on.

Common gotcha

It is a set of functions, not a widget you place — and you must import it: from tkinter import messagebox.

filedialogL2Level 3 lift

Let the user choose a file to open or save.

What it is

Dialogs from tkinter.filedialog that return the path the user picked — the GUI side of saving and loading.

When to reach for it

Opening or saving the user's own files, instead of always using a fixed filename.

Minimal example

import tkinter as tk
from tkinter import filedialog

root = tk.Tk()

def save():
    path = filedialog.asksaveasfilename(defaultextension=".txt")
    if path:                                  # "" if the user cancelled
        with open(path, "w", encoding="utf-8") as f:   # utf-8 keeps macrons safe
            f.write("Saved from my program")

tk.Button(root, text="Save as...", command=save).pack(padx=20, pady=20)
root.mainloop()

tk vs ttk

filedialog is a dialog module, not a themed widget — there is no ttk version. The dialogs are already native.

Key options

  • askopenfilename(...) — choose a file to open.
  • asksaveasfilename(defaultextension=, filetypes=).

Events & value

Each function returns a path string, or "" if the user cancels.

Common gotcha

Always check for the empty string (Cancel) before opening the path, or the program crashes when the user backs out. And open files with encoding="utf-8" so macrons (ā ē ī ō ū) read and write correctly.

Practice task

Level 3 lift

Pair filedialog with open(..., encoding="utf-8") inside a class to read and write the user's files — this is "reading from / writing to persistent storage", a named complex technique, and the natural partner to nearly every Level 3 project.

ToplevelL3

A second window.

What it is

An extra window on top of the main one — a settings screen, an "add item" dialog, an about box.

When to reach for it

Multi-screen programs, or a focused sub-task that deserves its own window.

Minimal example

import tkinter as tk

root = tk.Tk()

def open_settings():
    win = tk.Toplevel(root)
    win.title("Settings")
    tk.Label(win, text="Settings go here").pack(padx=20, pady=20)

tk.Button(root, text="Settings...", command=open_settings).pack(padx=20, pady=20)
root.mainloop()

tk vs ttk

Toplevel is tk-only — there is no ttk version. The widgets you put inside it can be tk or ttk.

Key options

  • tk.Toplevel(parent) — create the window.
  • transient(parent) — keep it above its parent.
  • grab_set() — make it modal (blocks the main window until closed).

Events & value

Build widgets in it like any window. Pass data back by updating the parent window from inside the Toplevel's handlers.

Common gotcha

Do not call tk.Tk() for a second window — that is only for the one main window. Extra windows are always Toplevel.

MenuL3

The menu bar across the top of a window.

What it is

Drop-down menus attached to the window — File, Edit, Help, and so on.

When to reach for it

Organising the commands of a larger program so the window stays uncluttered.

Minimal example

import tkinter as tk

root = tk.Tk()
root.geometry("300x150")

menubar = tk.Menu(root)
file_menu = tk.Menu(menubar, tearoff=0)
file_menu.add_command(label="Exit", command=root.quit)
menubar.add_cascade(label="File", menu=file_menu)
root.config(menu=menubar)

root.mainloop()

tk vs ttk

Menu is tk-only — there is no ttk.Menu. It already uses the native menu bar.

Key options

  • add_command(label=, command=), add_separator().
  • add_cascade(label=, menu=) attaches a drop-down.
  • tearoff=0 removes the dotted tear-off line.

Events & value

Each menu item has a command=, exactly like a Button.

Common gotcha

Build the bar, build each drop-down, add the cascades, then root.config(menu=bar). Forgetting the last step means no menu appears.

ttk.TreeviewL3

A multi-column table of records.

What it is

A widget (from ttk) that shows rows with several columns — where a Listbox shows only one.

When to reach for it

Displaying records or tabular data — a list of students with name, year, and grade.

Minimal example

import tkinter as tk
from tkinter import ttk

root = tk.Tk()
tree = ttk.Treeview(root, columns=("name", "year"), show="headings")
tree.heading("name", text="Name")
tree.heading("year", text="Year")
tree.insert("", tk.END, values=("Aroha", 12))
tree.insert("", tk.END, values=("Ben", 13))
tree.pack(padx=20, pady=10)
root.mainloop()

tk vs ttk

Treeview is ttk-only — there is no classic version. Import it with from tkinter import ttk.

Key options

  • columns=, show="headings".
  • heading(col, text=), column(col, width=, anchor=).
  • insert("", tk.END, values=(...)).

Events & value

selection() returns the selected row ids (can be empty); tree.item(id, "values") reads a row. React with <<TreeviewSelect>>.

Common gotcha

By default an empty first column appears — use show="headings" to hide it. And selection() can be empty, so check before indexing.

ScrollbarL3

Scroll a widget whose contents overflow.

What it is

A scrollbar wired to a Listbox, Text, Treeview, or Canvas so the user can scroll through more content than fits.

When to reach for it

Any list or text area that can grow past the size of its box.

Minimal example

import tkinter as tk

root = tk.Tk()
frame = tk.Frame(root)
frame.pack(fill="both", expand=True, padx=10, pady=10)

scrollbar = tk.Scrollbar(frame, orient="vertical")
listbox = tk.Listbox(frame, yscrollcommand=scrollbar.set)   # list -> scrollbar
scrollbar.config(command=listbox.yview)                     # scrollbar -> list
scrollbar.pack(side="right", fill="y")
listbox.pack(side="left", fill="both", expand=True)

for i in range(1, 51):
    listbox.insert(tk.END, f"Item {i}")
root.mainloop()

tk vs ttk

Both families have a Scrollbar, wired exactly the same way; ttk.Scrollbar looks more native.

Classic tk

tk.Scrollbar(frame, orient="vertical")

Themed ttk

from tkinter import ttk
ttk.Scrollbar(frame, orient="vertical")

Key options

  • orient="vertical" / "horizontal".
  • Wire both ways: yscrollcommand=sb.set and command=widget.yview.

Events & value

Handled entirely by the two-way wiring — there is nothing to read.

Common gotcha

It must be wired in both directions. Wire only one and the scrollbar looks right but does nothing.

Counts as: part of a usable, refined interface.
ttk.NotebookL3

A tabbed interface.

What it is

A widget (from ttk) that gives you tabs. Each tab is a Frame you fill with its own widgets.

When to reach for it

Organising a multi-section program in one window — a tidy alternative to several Toplevel windows.

Minimal example

import tkinter as tk
from tkinter import ttk

root = tk.Tk()
notebook = ttk.Notebook(root)

home = ttk.Frame(notebook)
tk.Label(home, text="Home tab").pack(pady=20)
notebook.add(home, text="Home")

about = ttk.Frame(notebook)
tk.Label(about, text="About tab").pack(pady=20)
notebook.add(about, text="About")

notebook.pack(fill="both", expand=True)
root.mainloop()

tk vs ttk

Notebook is ttk-only — there is no classic version. Import it with from tkinter import ttk.

Key options

  • ttk.Notebook(parent).
  • add(frame, text="...") adds a tab.

Events & value

Each tab is just a Frame. React to tab changes with <<NotebookTabChanged>> if you need to.

Common gotcha

Build and fill each Frame before you add() it. Tabs are Frames, not separate windows.

CanvasL3

A blank area for drawing shapes, lines, and text.

What it is

A drawing surface where you place shapes, lines, and text using pixel coordinates — for visual output a Label cannot give, like a diagram or a simple chart.

When to reach for it

Diagrams, simple charts, plotting positions, or displaying drawn graphics. This guide treats Canvas for display, not game mechanics.

Minimal example

import tkinter as tk

root = tk.Tk()
canvas = tk.Canvas(root, width=300, height=200, bg="white")
canvas.pack(padx=10, pady=10)
canvas.create_rectangle(20, 20, 120, 120, fill="#1D9E75")
canvas.create_text(150, 150, text="Hello")
root.mainloop()

tk vs ttk

Canvas is tk-only — there is no ttk.Canvas. Use tk.Canvas.

Key options

  • width, height, bg.
  • create_line, create_rectangle, create_oval, create_text.

Events & value

Coordinates start at the top-left: x grows right, y grows down. You build a drawing by adding items at computed positions.

Common gotcha

y increases downward, so y=0 is the top of the canvas — a common surprise when positioning items.
